# 项目部署到 Cloudflare Pages 说明

## 项目概述

这是一个基于 React + TypeScript + Vite + Tailwind CSS 的单页面应用（SPA），专门为 Cloudflare Pages 部署优化。

## 前置要求

1. GitHub/GitLab/Bitbucket 账号（用于代码仓库）
2. Cloudflare 账号（免费版即可）

## 部署步骤

### 1. 准备代码仓库

将项目推送到 GitHub/GitLab/Bitbucket 等 Git 托管平台。

### 2. 创建 Cloudflare Pages 项目

1. 登录 Cloudflare 控制台
2. 进入 **Workers & Pages** 页面
3. 点击 **Create application** → **Pages** → **Connect to Git**
4. 选择您的 Git 仓库并授权
5. 选择要部署的分支（通常是 `main` 或 `master`）

### 3. 配置构建设置

在 **Build settings** 部分配置如下：

- **Framework preset**: 选择 `Vite`
- **Build command**: `npm run build`
- **Build output directory**: `dist`
- **Root directory**: 留空（使用项目根目录）

### 4. 环境变量（如需要）

如果项目需要环境变量，请在 **Environment variables** 部分添加。

### 5. 部署

点击 **Save and Deploy** 开始部署。Cloudflare Pages 会自动：
- 拉取代码
- 安装依赖
- 执行构建
- 部署到全球 CDN

## Cloudflare Pages 配置说明

### _redirects 文件
位于 `public/_redirects`，用于 SPA 路由支持：
```
/*    /index.html   200
```
这确保所有路由请求都会返回 index.html，由前端路由处理。

### _headers 文件
位于 `public/_headers`，设置安全头和缓存策略：
- 安全头：X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy
- 静态资源缓存：`/assets/*` 设置为长期缓存（1年）

### Vite 配置优化
- `base: '/'` - 正确的资源路径
- 代码分割：将 React、React Router 等库单独打包
- 禁用 source map，减小构建体积
- 使用 esbuild 进行快速压缩

## Cloudflare 免费套餐限制

本项目完全符合 Cloudflare 免费套餐限制：

| 限制项 | 免费套餐 | 本项目情况 |

| 构建次数 | 500次/月 | 按需构建 |
| 构建时间 | 20分钟/次 | 约1-2分钟 |

## 自定义域名（可选）

1. 在 Cloudflare Pages 项目设置中，进入 **Custom domains**
2. 点击 **Set up a custom domain**
3. 输入您的域名并按照提示完成 DNS 配置
4. Cloudflare 会自动为您配置 SSL 证书

## 持续部署

每次推送到配置的分支，Cloudflare Pages 会自动触发新的构建和部署。

## 故障排除

### 构建失败
- 检查 `package.json` 中的构建脚本
- 确保所有依赖都正确声明
- 查看 Cloudflare Pages 的构建日志

### 路由 404 错误
- 确认 `_redirects` 文件存在于 `public/` 目录
- 确认重定向规则正确

### 样式或资源加载失败
- 检查 `vite.config.ts` 中的 `base` 配置
- 确认资源路径正确
